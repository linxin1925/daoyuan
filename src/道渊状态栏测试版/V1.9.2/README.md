## V1.9.2 相对 V1.9.0 的变更

### 1. 标签数量徽标

每个标签名右侧显示金色数量徽标，数据源与对应面板内部计数一致：

| 标签 | 计数来源 |
| --- | --- |
| 功法 | `stat_data.主角.功法` |
| 人物 | `stat_data.人物` |
| 灵宠 | `stat_data.灵宠` |
| 道侣 | `stat_data.道侣` |
| 机遇 | `stat_data.机遇` |
| 绝色榜 | `stat_data.绝色榜` |

| 储物 | `stat_data.主角.储物袋` |
| 器物 | `stat_data.主角.器物` |

副业、地图、宗门不显示数量（非列表型内容）。

### 2. 特殊立绘持久化

**问题**：特殊立绘选中状态只在立绘弹窗组件内存中，关弹窗即丢失。其他三种立绘（自定义/多套/女版）都有 localStorage 持久化。

**修复**：
- 新增 `daoyuan_special_portraits` 存储键（`{角色名: true}`），带 `·/・` 分隔符归一化
- 全局立绘解析函数 `vt` 插入特殊立绘分支，优先级：**自定义 > 特殊（已持久化）> 女版 > 多套立绘**——人物/道侣/灵宠/绝色榜所有面板缩略图都显示特殊立绘
- 弹窗打开时从 localStorage 恢复特殊立绘状态

"最后一次选择即持久显示"规则：

| 操作 | 持久化行为 |
| --- | --- |
| 点击"查看特殊立绘" | ✅ 写入持久化，所有地方都显示特殊立绘 |
| 点击"当前：特殊立绘（点击切回）" | ❌ 手动取消，清除持久化 |
| 点选多套立绘缩略图（更换立绘） | ❌ 清除特殊持久化，记住新索引 |
| 切换女版立绘 | ❌ 清除特殊持久化，记住女版 |
| 上传本地图片 / 粘贴 URL（自定义） | ❌ 清除特殊持久化，自定义优先级最高 |
| 关闭弹窗 / 刷新 / 切换楼层 | ✅ 保持 |

### 3. 立绘公告按钮

顶栏右侧、齿轮按钮左边新增卷轴图标按钮"立绘公告"。点击后远程拉取 `notice.json` 显示公告弹窗：

- 数据源：`cdn.jsdelivr.net/gh/YttriumCarbide/Daoyuan@main/notice.json`（回退 `raw.githubusercontent.com`）
- 弹窗标题自动拼接 `version` 和 `date`
- 遍历 `tabs` 字段，每个 key 渲染为一节（金色左边线小标题 + 正文，`\n` 换行保留）
- 会话内已加载则复用，不重复请求

### 4. 玉简未读消息提醒（微信式）

**两级红色未读徽标**：
1. 玉简标签：标签名右边显示红色圆形未读总数（所有联系人未读之和，99+ 封顶）
2. 联系人列表：每个联系人行右侧显示各自的红色未读数

**判定逻辑**：
- 未读 = 历史记录中发送者不是"我"的消息条数 - localStorage 已读水位线
- 持久化键 `daoyuan_yujian_read`（`{角色名: 已读条数}`）

**标记已读时机**：
- 点开某个联系人的聊天界面 → 该联系人清零
- 正停留在聊天界面时对方新回复到达 → 自动即时清零
- 只在列表外看到徽标但没点进去 → 未读保持

AI 回复更新变量后（MVU 事件刷新数据），未读数自动重算。
