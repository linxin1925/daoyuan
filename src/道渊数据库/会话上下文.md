# 道渊 v5.2 数据库填表项目 · 会话上下文

## 项目目标
为《道渊》v5.2 角色卡编写「星河璀璨·数据库」的填表预设和填表提示词。
- 脚本：SP·数据库 V（`shujuku_v120`，源码 `https://gcore.jsdelivr.net/gh/AlbusKen/shujuku@spv7.0/index.js`），SQLite 引擎，AI 每轮自动填表

- 模式：normal（修真养成），非交火模式

## 关键设计决策

1. **过往经历**：每轮覆盖更新（≈100字近期摘要），不累积拼接。完整历史由修真纪要表维护
2. **物品当前状态**：标记物品归属（主角储物袋中/寄拍于万宝楼/已赠予XX/待移交XX），不依赖 DELETE
3. **已删除选项表**，从 8 张减到 7 张
4. **功法技能列**（重要角色表 v5.2 新增）：记 NPC 功法/剑诀/神通/法术/血脉天赋/本能能力名称，多项「、」分隔，仅记名不记境界。世界书格式不统一时从中提取并归一；性格/外貌/临时增益不计入；未明文填「（未提及）」
5. **inventory 去 CHECK 约束**（V0.2 已改，Deepseek 待同步）：原 `CHECK(quantity > 0)` 跟「已赠予物品保留 quantity=0 + current_status 标记」设计冲突。V0.2 已去掉 CHECK，quantity=0 仅用于已赠予/已失去但保留记录的物品；消耗/损毁才 DELETE
6. **INSERT 已存在实体必须 UPDATE**（V0.2 已改）：带 UNIQUE 列的表（important_characters.name / inventory.item_name / protagonist_skills.skill_name / quests_events.quest_name）已存在的实体必须 UPDATE，禁止 INSERT（会 UNIQUE 冲突）。INSERT 段统一为「省略 row_id，让 SQLite 自动递增」
7. **MVU 与数据库分工**：MVU 管实时状态+精确增删，数据库管长期归档+结构化查询
8. **V0.3 删除任务与事件表**（2026-07-04）：该表易被 MVU `stat_data.机遇` 干扰、记录无用信息（主线/支线任务进度），V0.3 移除。MVU 仍保留机遇变量，数据库不再归档任务。提示词/预设/说明文档同步清理（UNIQUE 列示例、DELETE 段、每表填写要点、章节编号）。V0.2-Deepseek 未同步

## 数据库注入原理

- **填表回合**：脚本通过 `$0` 注入全量 DDL + 当前数据到提示词
- **非填表回合**：重要角色表和纪要表通过 `exportConfig` 注入世界书（关键词触发）
- **AI 输出**：SQL 文本（`<tableEdit>`，V0.2）或严格 JSON（V0.2-Deepseek）
- **数据存储**：sql.js WASM 内存，非 IndexedDB

## 严格 JSON 模式（Deepseek 版）

脚本「严格 JSON 填表响应」开关（`strictJsonTableFillEnabled`）开启后，填表改用 `strictJsonSqlCharCardPrompt` 槽位（SQLite 模式）而非 `charCardPrompt`。

- **JSON schema**：`{"format":"table_edit_sql_v1","sql":"<SQL 脚本>"}`（源码 `buildStrictJsonSqlPrompt_ACU`，spv7.0）。仅 `format`+`sql` 两字段，无 thought 字段，AI 内部推理不输出
- **API 参数**（预设「附加主体参数」custom_include_body）：`response_format: {type: json_object}` + `thinking: {type: disabled}`
- **提示词差异**：输出格式段改 JSON 外壳；SQL 格式要点加 JSON 转义规则（`\n`/`\"`）；assistant 段改「只返回 JSON 对象」
- **转义强化**：加 ⚠️ 单引号转义警告 + 正反例；加「中文引号「」规则」——字符串值内部对话/暗语/招式名引用必须用「」包裹，禁止单引号（避免 SQL 转义）和双引号（避免 JSON 转义）
- **导入坑**：导入提示词必须**先开严格 JSON 开关再导入保存**，否则会存错槽位（存到 charCardPrompt 而非 strictJsonSqlCharCardPrompt）。脚本根据开关决定存哪个槽位（`getCurrentPromptSettingKey_ACU`）
- **应用预设坑**：导入表格预设会存到模板预设库（`templatePresets`），需在面板「应用/加载」为当前 template（`currentTemplatePresetName`）才生效；光导入不应用，脚本仍用旧 template

## 已知问题（Deepseek-v4-flash 限制，200楼回填压力场景）

- **截断**：flash 输出上限低（配 max_tokens 60000 但 flash 实际上限远低于此），长 chronicle INSERT 容易被截断（`incomplete input`，缺 `);`）。日常几层楼一次不会触发，200楼批量回填明显
- **转义遵循度低**：flash 在长 chronicle 里漏单引号转义（`'对话'` 不转义），即使提示词有警告 + 中文引号规则。日常轻负担下好很多
- **JSON 格式混淆**：偶发把 SQL 结尾 `);` 写成 `}`（`unrecognized token: "}"`）
- **应对**：靠脚本失败重填机制兜底；分批填表（调小 `updateBatchSize`）降低单次负担；不换模型（v4 成本高，flash + 缓存命中成本最优）
