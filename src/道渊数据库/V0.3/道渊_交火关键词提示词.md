# 道渊 V0.3 · 交火模式关键词生成提示词（修真世界观专用）

> 基于《道渊》v5.2 修真世界观定制的交火模式关键词生成提示词。
> 替换脚本「☰星河璀璨·数据库」交火模式设置面板里的 **关键词生成提示词**（`keywordPromptGroup`）。
> 适配 V0.3 纪要表结构（time_span / location / chronicle_text / summary / code_index）。
> 源码机制见 [../交火模式源码分析.md](../交火模式源码分析.md) 第五节。

---

## 一、设计说明

源码默认提示词的关键词优先级是通用维度（人物、地点、时间、事件、目标、冲突、道具、组织、关系变化、未解决问题）。道渊专用版针对修真世界观重构优先级，新增修真专属检索维度：

| 维度 | 修真专属要点 |
|---|---|
| 境界 | 炼气/筑基/金丹/元婴/化神/合体/大乘/渡劫 + 阶段（初期/中期/后期/大圆满）|
| 功法气运 | 功法/剑诀/神通/气运全名（保留「真·」前缀）|
| 宗门 | 宗门归属 + 弟子身份（记名/真传/亲传）|
| 气运机缘 | 气运开启/消耗/进阶（初识/小成/大成/圆满）|
| 物品 | 修真物品分类（灵石/丹药/灵草/法器/法宝/符箓/玉简/材料）|

并强制使用修真标准术语（如「金丹初期」而非「刚结金丹」），提升向量召回的语义一致性。

---

## 二、提示词内容（三条消息）

### 消息 1 · system（`deletable: false`）

```text
你负责为《道渊》v5.2 修真世界观下的交火模式纪要索引召回生成检索关键词。
你会看到最近对话上下文和当前用户输入（主角赤羽视角的修真剧情）。
目标：输出最相关的 12 个简洁关键词或短语，用于纪要表（chronicle）向量索引的召回与重排序。

优先级（修真世界观维度，按重要性排序）：
- 人物：修仙者姓名/道号/称呼（如 赤羽、晏青、红蝶、温月清）
- 地点：修真地理位置，按 大区·势力·场景 格式（如 中央神州·湮丹宗·杏临谷、烬骨荒原、万宝楼）
- 境界：修仙境界及阶段（如 炼气期、筑基初期、金丹中期、元婴大圆满、化神、渡劫）
- 功法气运：功法/剑诀/神通/气运名称，保留「真·」前缀（如 九阳神功、草字剑诀、真·时间掌控者、真·诸天万界钓鱼）
- 宗门：宗门归属与弟子身份（如 湮丹宗·记名弟子、散修、明月宗·真传弟子、宗主亲传）
- 事件：修真事件类型（如 突破、夺宝、论道、渡劫、钓鱼执法、寄拍、机缘奇遇、采药、炼丹、赠予）
- 物品：法宝/丹药/灵草/玉简/材料（如 陨铁剑胚、淬体丹、紫灵米、下品灵石、储物袋、玄铁飞剑）
- 冲突：争斗与对抗（如 魔修、夺舍、心魔反噬、围剿、斗法、仇怨）
- 关系变化：人际变动（如 师徒、道侣、盟友、赠予、引荐、背叛、好感提升）
- 气运机缘：气运开启/消耗、机缘触发、气运进阶（初识/小成/大成/圆满）
- 未解决问题：悬而未决的线索、未完成的机缘、未解之谜、待移交物品、待办承诺

【输出格式 — 必须严格遵守】
你的回复必须且只能包含以下两部分，不得使用其他任何格式：

<thinking>
逐步分析最近上下文与当前用户输入，识别涉及的修真要素：人物、地点、境界、功法气运、宗门、事件、物品、冲突、关系变化、气运机缘和未解决问题。
</thinking>
<keywords>关键词1，关键词2，关键词3</keywords>

【硬性规则】
- 关键词必须且只能放在 <keywords></keywords> 标签之间。
- 禁止使用"关键词："前缀输出，禁止使用编号或列表格式。
- <keywords> 标签内只放关键词或短语，不要放解释句、编号、前后缀说明。
- 多个关键词必须使用中文逗号分隔。
- 尽量输出 12 个，最多 24 个。
- 优先使用修真世界观的标准术语：
  · 境界用标准阶位（"金丹初期"而非"刚结金丹"）
  · 地点用修真地理全名（"湮丹宗·杏临谷"而非"那个药田"）
  · 功法气运保留完整名（"真·草字剑诀"含前缀）
  · 人物优先用道号/本名
- <keywords> 标签外的任何内容都不会被用于检索匹配。
```

## 三、使用方法

### 方法 A：UI 手动替换（推荐）

1. 打开酒馆扩展脚本「☰星河璀璨·数据库」设置面板
2. 进入 **交火模式** 配置区
3. 找到 **关键词生成提示词** 编辑器（`keywordPromptGroup`）
4. 按顺序替换三条消息：
   - **system**：粘贴上方「消息 1」内容，`deletable` 设为 `false`
   - **user**：粘贴「消息 2」内容，`deletable` 设为 `true`
   - **assistant**：填 `<thinking>\n`，`deletable` 设为 `true`
5. 保存设置

### 方法 B：chrome-devtools MCP 注入

> 注入前先确认 `keywordPromptGroup` 的实际存储路径：

```js
const ST = window.SillyTavern.getContext();
const settings = ST.extensionSettings.__userscripts.shujuku_v120__userscript_settings_v1;
console.log('vectorMemoryConfigGlobal:', settings?.vectorMemoryConfigGlobal);
console.log('keywordPromptGroup:', settings?.vectorMemoryConfigGlobal?.keywordPromptGroup);
```

确认路径后注入（以下假设路径为 `settings.vectorMemoryConfigGlobal.keywordPromptGroup`）：

```js
const ST = window.SillyTavern.getContext();
const settings = ST.extensionSettings.__userscripts.shujuku_v120__userscript_settings_v1;
const config = settings.vectorMemoryConfigGlobal || (settings.vectorMemoryConfigGlobal = {});
config.keywordPromptGroup = [
  {
    role: 'system',
    content: '<<完整粘贴「消息 1 · system」内容>>',
    deletable: false,
  },
  {
    role: 'user',
    content: '最近上下文：\n$RECENT_CONTEXT\n\n当前用户输入：\n$USER_INPUT\n\n请根据以上内容生成交火模式纪要索引召回关键词。先在 <thinking> 中分析，然后在 <keywords></keywords> 标签中输出关键词。',
    deletable: true,
  },
  {
    role: 'assistant',
    content: '<thinking>\n',
    deletable: true,
  },
];
ST.saveSettingsDebounced?.();
```

## 四、与默认提示词的差异

| 项 | 默认提示词 | 道渊专用版 |
|---|---|---|
| 优先级维度 | 10 项通用（人物/地点/时间/事件/目标/冲突/道具/组织/关系变化/未解决问题）| 11 项修真专属（新增 境界/功法气运/宗门/气运机缘，细化 物品/事件/冲突）|
| 术语规范 | 无 | 强制修真标准术语（境界阶位、地理全名、功法全名含前缀）|
| 示例 | 无 | 每个维度附道渊实际术语示例（赤羽/湮丹宗/金丹初期/真·草字剑诀等）|
| 输出格式 | `<thinking>` + `<keywords>` | 同（保持兼容解析）|

输出格式与解析逻辑（`parseKeywords_ACU`）完全兼容，不影响脚本运行。

## 五、配套配置建议

| 配置项 | 建议值 | 理由 |
|---|---|---|
| `keywordContextPairCount` | 2-3 | 修真剧情上下文跨度大，多读几层有助于提取人物/功法线索 |
| `keywordGenerationMaxAttempts` | 3（默认）| 修真术语多，重试有助稳定输出 |
| `keywordApiPreset` | 视成本定 | 可用便宜模型生成关键词（如 Gemini Flash），与主填表模型分离 |
| `summaryIndexKeywordMinRows` | 200（默认）| 纪要累积 200 条后召回性价比高 |
| `embeddingModel` | 中文优化模型 | 修真术语多中文专有词，中文 embedding 召回更准 |
