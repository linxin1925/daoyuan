## 1. 机制总览

### 1.1 ST 宿主层（非卡片作者控制）

SillyTavern 宿主在移动端的行为由五个子机制决定。

#### A. 视口声明（index.html）

```html
<meta name="viewport" content="width=device-width, viewport-fit=cover,
  initial-scale=1, maximum-scale=1.0, user-scalable=no,
  interactive-widget=resizes-content">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="mobile-web-app-capable" content="yes">
```

`user-scalable=no` 禁止用户手势缩放；`interactive-widget=resizes-content` 使虚拟键盘弹出时 viewport 收缩而非仅覆盖内容，整体布局随之压缩。

**来源：** ST v1.14.0 `public/index.html` | **置信度：high** `[运行时源码]`

---

#### B. 移动端主断点：1000px

ST 在 `mobile-styles.css` 里以 `@media (max-width: 1000px)` 为核心断点做全面 reflow：

- 所有侧边栏切换为 `width: 100dvw; max-width: 100dvw; position: fixed; left: 0`（覆盖全屏抽屉）
- 面板 pin 按钮隐藏（`display: none`）
- body 固定：`touch-action: none; overflow: hidden; position: fixed`
- 导航面板强制 `min-width: 100dvw !important`
- 次断点 `@media (max-width: 450px)` 管窄屏手机（如 320px 竖屏）额外调整

**来源：** ST v1.14.0 `public/css/mobile-styles.css` | **置信度：high** `[运行时源码]`

### 1.2 卡片层（角色卡作者可控）

卡片的 HTML 内容（通过正则扩展替换）被注入到 `.mes_text` 内。ST v1.14.0 源码中**不存在由 ST 框架层创建 iframe 的逻辑**（已 grep 全部 .js 文件确认）；卡片内的 `<iframe>` 是卡片自身 HTML 创建的（如挂载壳自含 `<iframe id="xy-sb-inner">`）。这个卡片自创的 iframe 在 ST 聊天气泡内运行，其 `window.frameElement` 返回的是 ST `.mes_text` 内卡片 HTML 自己的外层 iframe 元素，从而卡片 JS 可控制其高度。**ST 框架层不施加 `sandbox` 属性**（源码层无该逻辑），卡片 iframe 可自由访问 `document`。

**来源：** 本地实战生产卡 3.0.2（status_bar_regex.html 实际代码）+ ST v1.14.0 全脚本 grep 确认 | **置信度：medium·单源孤证** `[运行时源码]`

> **校准备注（2026-06-28）：** 原描述「ST 以 iframe 形式渲染」有误——是卡片 HTML 自含 iframe，不是 ST 框架创建。sandbox 问题（原悬案 U2）经源码 grep 可排除「ST 主动施加 sandbox」，但浏览器自身对 blob: 来源的限制行为未做真机测试，保留 U2 供验证。

## 3. 完整可用范式代码

### 3.1 iframe 高度自适应（星月 3.0.x 生产用法）

**来源：** `星月/星月 3.0.2/components/status_bar_regex.html` | **置信度：high** `[运行时源码]`

机制：外层「挂载壳」fetch 远程真实 HTML → Blob URL 挂到内层 iframe → ResizeObserver 持续同步高度到 ST 控制的最外层 iframe。

```html
<!-- 挂载壳 body -->
<style>
  html, body { margin: 0; padding: 0; overflow: hidden; }
  #xy-sb-inner {
    display: block;
    width: 100%;
    border: 0;
    overflow: hidden;
  }
</style>

<iframe id="xy-sb-inner" scrolling="no"></iframe>

<script>
(function () {
  var inner = document.getElementById('xy-sb-inner');

  /* ── 高度同步：把内层 iframe 真实高度同步到 ST 外层 iframe ── */
  function syncHeight() {
    try {
      var fe = window.frameElement;   // ST 控制的外层 iframe 元素
      if (!fe) return;
      var doc = inner.contentDocument ||
                (inner.contentWindow && inner.contentWindow.document);
      if (!doc) return;

      /* 量高前禁横向溢出，避免横向滚动条撑高 */
      try {
        if (doc.body) {
          doc.body.style.overflowX = 'hidden';
          doc.body.style.maxWidth  = '100%';
        }
      } catch (_) {}

      var h = Math.max(
        doc.documentElement ? doc.documentElement.scrollHeight : 0,
        doc.body            ? doc.body.scrollHeight            : 0
      );

      if (h > 0 && Math.abs((parseInt(fe.style.height, 10) || 0) - h) > 2) {
        fe.style.height    = h + 'px';
        fe.style.minHeight = h + 'px';
        inner.style.height = h + 'px';
        try { fe.setAttribute('height', String(h)); } catch (_) {}
      }
    } catch (_) {}
  }

  function mountInner(html) {
    var blob    = new Blob([html], { type: 'text/html' });
    var blobUrl = URL.createObjectURL(blob);

    inner.addEventListener('load', function onLoad() {
      inner.removeEventListener('load', onLoad);
      syncHeight();
      // 异步渲染补偿（内部脚本延迟填充内容）
      [100, 400, 900, 1800, 3000].forEach(function (t) {
        setTimeout(syncHeight, t);
      });
      // ResizeObserver 持续同步
      try {
        var doc = inner.contentDocument ||
                  (inner.contentWindow && inner.contentWindow.document);
        if (doc && doc.body) new ResizeObserver(syncHeight).observe(doc.body);
      } catch (_) {}
    });

    inner.src = blobUrl;
  }

  // 顺序尝试主 CDN → 备源 → fail
  fetch('https://cdn.example.com/status-bar.html')
    .then(function (r) { if (!r.ok) throw new Error('HTTP ' + r.status); return r.text(); })
    .then(function (html) { mountInner(html); })
    .catch(function () { /* 备源逻辑 */ });
})();
</script>
```

**关键要点：**

- 量高前先强制 `overflowX: hidden; maxWidth: 100%`，避免横向滚动条干扰 scrollHeight
- 容差 `> 2px` 防止高频无意义 DOM 写入
- 多个延迟 `setTimeout([100, 400, 900, 1800, 3000])` 补偿内部异步渲染（动画/数据填充）
- `ResizeObserver` 兜底持续同步，覆盖内容动态变化

### 3.2 响应式 CSS 变量设计系统（星月 2.5.0 生产用法）

**来源：** `星月/星月 2.5.0/components/status_bar_regex_beauty_r5.html` | **置信度：high** `[运行时源码]`

核心思路：**所有尺寸锚定到一个根变量 `--hud-base-font`**，用户通过 settings slider 调整后自动缩放所有元素。

```css
:root {
  --hud-base-font:  12px;   /* 锚点；用户可通过设置滑块调节 */
  --hud-base-width: 100%;
  --cut: 10px;              /* 切角尺寸 */
}

body {
  margin: 0; padding: 0;
  font-size: var(--hud-base-font);
}

/* 容器：宽度 100% 不设固定 max-width，由用户设置的 widthPercent 动态注入 */
.hud-container {
  width: 100%;
  max-width: var(--hud-base-width);
  margin: 0 auto;
  box-sizing: border-box;
}

/* 字体比例关系（全部用 calc 相对于锚点） */
.hud-title    { font-size: calc(var(--hud-base-font) * 1.64); }
.hud-subtitle { font-size: calc(var(--hud-base-font) * 0.82); }
.hud-tag      { font-size: calc(var(--hud-base-font) * 0.92); }
.settings-note { font-size: calc(var(--hud-base-font) * 0.86); }
```

JS 端动态注入（响应用户调节）：

```javascript
var hudSettings = { fontSize: 12, widthPercent: 100 };

function applyHudSettings(s) {
  var root = document.documentElement;
  root.style.setProperty('--hud-base-font',  s.fontSize + 'px');
  root.style.setProperty('--hud-base-width', s.widthPercent + '%');
}

// 绑定 slider
document.getElementById('opt-fontsize').addEventListener('input', function () {
  applyHudSettings({ fontSize: +this.value, widthPercent: hudSettings.widthPercent });
});
```

JS 端弹出框定位（防止溢出屏幕右/下边缘）：

```javascript
function positionPopover(anchor, $popover) {
  var rect = anchor.getBoundingClientRect();
  var pw   = $popover.outerWidth()  || 310;
  var ph   = $popover.outerHeight() || 260;
  var left = Math.max(8, Math.min(window.innerWidth - pw - 8, rect.left));
  var top  = rect.bottom + 6;
  // 下方空间不足时翻转到上方
  if (top + ph > window.innerHeight - 8) top = Math.max(8, rect.top - ph - 6);
  $popover.css({ left: left + 'px', top: top + 'px' });
}
// 窗口 resize 时重新定位
$(window).on('resize', function () { positionPopover(anchor, $popover); });
```

### 3.5 触控事件正确绑定

$picker.on('pointerdown', '.picker-stage', function (e) {

### 3.6 动态网格宽度检测（容器查询替代方案）

**来源：** `status_bar_regex_beauty_r5.html` 第 2474–2477 行 | **置信度：high** `[运行时源码]`

```javascript
// 用 getBoundingClientRect 做运行时响应（以容器宽度判断，而非 viewport 宽度）
function updateMediaLayoutBlocks() {
  document.querySelectorAll('.cz-media-block').forEach(function (el) {
    var width = el.getBoundingClientRect().width;
    el.classList.toggle('is-narrow', width > 0 && width < 560);
  });
}
// 在 window resize 时调用
window.addEventListener('resize', updateMediaLayoutBlocks);
```

此方案可规避 iframe 宽度 vs window 宽度的歧义，直接测量目标元素实际渲染宽度。

## 4. 高频避坑

### P1 iframe 高度在移动端卡 0 或撑不开

**原因：** 量 `scrollHeight` 时若有横向溢出，`body.scrollHeight` 会包含溢出宽度导致计算异常；移动端系统字体放大时 layout 可能延迟完成。

**解法（已验证）：** 量高之前强制 `doc.body.style.overflowX = 'hidden'; doc.body.style.maxWidth = '100%'`，多轮 `setTimeout [100, 400, 900, 1800, 3000]` + `ResizeObserver` 持续跟踪。

**来源：** 本地实战源码 3.0.2 | **置信度：high** `[运行时源码]`

---

### P2 `100vh` 在移动端 Safari 比实际可用高度高

**原因：** `100vh` 包含被浏览器工具栏遮挡的区域（Safari 底部工具栏约 56–80px）。

**解法：** ST 宿主已在 `mobile-styles.css` 改用 `100dvh`（dynamic viewport height）。卡片内部弹出框用 `min(N px, calc(100vh - 偏移))` 兜底；若需精确贴合，改用 `100dvh`。

**来源：** 运行时源码（ST 使用 `100dvh` 已确认）+ 推断（卡片内解法未真机验证，Safari 工具栏高度为社区通识） | **置信度：high（ST 宿主用 dvh 部分）** + **medium·推导未验证（卡片内 dvh 应用建议）** `[运行时源码]` `[推断]`

---

### P3 固定宽度 px 值在窄屏被截断

**原因：** 状态栏面板 `width: 328px` 在 320px 宽手机上溢出屏幕右侧。

**解法（已验证）：** `width: min(328px, calc(100vw - 24px))`，`24px` 为两侧各留 12px 安全边距。

**来源：** 本地实战源码 2.5.0 | **置信度：high** `[运行时源码]`

### P4 媒体查询在 iframe 内以 iframe 宽度为参照而非 window

**原因：** iframe 内的 `@media (max-width: 600px)` 响应 iframe 自身渲染宽度，不是宿主 window 宽度（Web 标准行为）。

**影响（正向）：** 即使宿主 window 是大屏，只要 iframe 被 ST 聊天气泡约束到窄宽度，媒体查询也会正确触发；卡片内断点可直接以气泡/消息容器宽度为设计基准。

**注意：** ST 是否对 iframe 施加 `sandbox` 属性覆盖此行为尚未完整验证（见 6 悬案 U2）。

**来源：** Web 标准规范 + 推断 | **置信度：medium** `[推断]`（需真机验证 sandbox 模式）

### P7 `position: fixed` 在 iframe 内相对于 iframe 视口而非宿主 window

**原因：** iframe 建立独立 stacking context，`fixed` 相对 iframe 视口定位，而非宿主页面视口。

**影响：** 卡片内 `position: fixed` 的弹出框/设置面板在移动端可能被卡片自身 iframe 的高度限制截断（iframe 高度由 syncHeight 同步到内容高度，弹出框若超出可视区则被裁切）。注意：ST 框架层不直接限制 iframe 高度，高度由卡片自身 JS 控制。

**解法（已验证）：** 改用 `position: absolute` + 父容器 `position: relative; overflow: visible`；或动态检测下方剩余空间，不足时翻转弹出方向（见 3.3 JS 定位代码）。

**来源：** Web 标准 + 本地实战源码翻转逻辑 | **置信度：high** `[运行时源码]` `[推断]`

### P8 移动端卡片内无法调用 ST 内部函数 `isMobile()`

**原因：** `isMobile()` 以 `export function` 形式定义在 ST 宿主 `RossAscends-mods.js` 中（已源码确认），属于宿主 ES module scope，卡片内 iframe JS 无法通过 `window.isMobile()` 直接调用（不同 document 上下文且 module export 不挂载到 window）。

**替代方案：**

```javascript
// 方案 A：视口宽度判断
var isMobileLayout = window.innerWidth < 1000;

// 方案 B：媒体查询对象
var isMobileLayout = window.matchMedia('(max-width: 600px)').matches;

// 方案 C：UA 检测（粗略）
var isMobileUA = /Mobi|Android|iPhone|iPad/i.test(navigator.userAgent);
```

**来源：** 运行时源码（isMobile 定义位置已确认）+ 推断（iframe 跨 scope 不可访问，理论层面无真机实测） | **置信度：medium·推导未验证** `[运行时源码]` `[推断]`

### P9 横竖屏切换时 iframe 高度不更新

**原因：** 设备旋转后视口宽高互换，内容重排，但若没有监听 resize/orientation 事件，iframe 高度停留在旋转前的值。

**解法：**

```javascript
window.addEventListener('resize', syncHeight);
if (screen.orientation) {
  screen.orientation.addEventListener('change', function () {
    setTimeout(syncHeight, 300); // 旋转动画完成后再量
  });
}
```

**来源：** 推断 | **置信度：medium** `[推断]`（需真机验证）
