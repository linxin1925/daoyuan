### 2.2 控制中心执行上下文与双窗口穿透

**核心结论**

ST 里的卡内 JS（通过正则脚本注入）运行在消息渲染 iframe 内。控制中心采用**双窗口穿透**策略——始终优先访问 `window.parent`（ST 主窗口），失败则回落到 `window` 自身。真实 DOM（扩展菜单栏、聊天输入框等）在父窗口中，必须通过 `hostWindow()` / `hostDocument()` 访问。

```javascript
// 来自 control_center.js v2.9.9 第 49-54 行（最高可信）
function hostWindow() {
  try {
    if (window.parent && window.parent !== window && window.parent.document) return window.parent;
  } catch (_) {}
  return window;
}
function hostDocument() {
  try { return hostWindow().document || document; } catch (_) { return document; }
}
```

**核心原则**：所有需要访问 ST 主窗口 DOM 或全局 API 的操作，均通过 `hostWindow()` / `hostDocument()`，不直接用 `window` / `document`。

详见《C1_前端基础-TavernHelper与iframe.md》§10（window.parent 防御访问模式）。

来源：`control_center.js` v2.9.9 第 49-54 行 ｜ 置信度：high `[运行时源码]`

### 2.5 入口层、外壳层与内容层选择

2026-07-14 起，状态栏/控制中心按三层拆分，避免把“按钮长什么样”“面板在哪里”和“状态怎么读”混成一份巨型实现：

```text
入口层：Wand / 悬浮球 / 页面按钮
                 ↓ command registry
HUD Host Core：phase / generation / abort / contextRevision / dirty
                 ↓ ShellAdapter
             移动抽屉 / 桌面浮窗
                 ↓
     iframe + bridge + 状态栏内容与数据链

消息楼层生命周期 → 每楼/末楼 inline iframe → 同一内容协议/数据适配
```

| 选择 | 适合 | 不适合 | 专册 |
|---|---|---|---|
| 消息内联状态栏 | 每楼或末楼紧贴剧情，天然跟随消息 | 需要长期悬浮、跨楼操作 | 本册 |
| Wand 入口 | 最稳妥的既有功能菜单入口 | 想常驻屏幕和拖动 | E5 |
| 悬浮球入口 | 常驻、可吸边、多功能分发 | 直接承载完整状态内容 | C11 |
| 抽屉外壳 | 窄屏、粗指针、软键盘环境 | 任意拖动/缩放 | C12 |
| 浮窗外壳 | 桌面边聊边看、可拖动缩放 | 手机默认方案 | C13 |

内容层仍由本册负责：变量读取、事件刷新、Tab/分区渲染、写回与控制中心业务。C12/C13 应复用同一个顶层 HUD Host Core 与 iframe 真身，只替换 ShellAdapter；消息内联通常是每楼/末楼自己的 iframe 生命周期，只共享内容协议和数据适配，不默认加入同一个活动 singleton session。C11 的入口只 dispatch 命令，不复制业务处理器。

星月 3.4.9 已把抽屉与浮窗纳入共享 `hudSession`。抽屉是已落地的移动端生产路线，不得因旧模块化 fragment 未同步而标成待验证。具体视口、手势、几何和异步竞态分别由 C12/C13 维护。

### 3.3 statRoot 兼容层与 safeGet 范式

**核心结论**：生产卡统一使用 `statRoot` 处理两种传入格式，`safeGet` 自动剥离 `stat_data.` 路径前缀。

```javascript
// stat_data 兼容层：允许传入完整 MvuData 或裸 stat_data
function statRoot(all) {
  if (!all || typeof all !== 'object') return {};
  return all.stat_data || all;
}

// 安全路径读取：自动剥离 'stat_data.' 前缀 + 空值处理
function safeGet(obj, path, def) {
  var root = statRoot(obj);
  var p = String(path || '');
  if (p.indexOf('stat_data.') === 0) p = p.slice(10);  // 兼容带前缀的 path
  var val = _.get(root, p);
  if (!shouldRenderValue(val)) return def;
  return displayValue(val, def);
}
```

**调用惯例**：外部调用建议统一传不带前缀的路径（如 `'user.等级'`），两种格式均支持。

来源：`status_bar_regex_beauty_r5.html` L1916-1918, L2324-2330 ｜ 置信度：high `[运行时源码]`

### 4.3 跨 iframe API 发现链（控制中心）

**核心结论**：卡内代码不能直接 `import` ST 扩展 API，只能通过全局对象发现。每个外部依赖走"本窗口 → 父窗口"双检。

```javascript
// Tavern Helper（核心依赖）
function helperHost() {
  const candidates = [window, hostWindow()];
  for (const target of candidates) {
    if (target?.TavernHelper) return target.TavernHelper;
    if (target?.generateRaw || target?.getChatMessages) return target;
  }
  return null;
}

// MVU（变量读写）
function mvuHost() {
  return window.Mvu || hostWindow().Mvu || null;
}

// 媒体库（同级模块）
function mediaLibrary() {
  return window.XingyueMediaLibrary || hostWindow().XingyueMediaLibrary ||
         window.CrossedZoneMediaLibrary || hostWindow().CrossedZoneMediaLibrary || null;
}
```

来源：`control_center.js` v2.9.9 ｜ 置信度：high `[运行时源码]`

### 4.4 TavernHelper 扩展 API（控制中心专用）

**核心结论**

| API | 用法 | 说明 |
|---|---|---|
| `TavernHelper.generateRaw(opts)` | `async → string` | 静默生成，不影响 chat history |
| `TavernHelper.getChatMessages(range)` | `→ Message[]` | 获取消息楼层，`-1` = 最新楼，整数 = 楼层 id |
| `Mvu.getMvuData({type:'message', message_id:'latest'})` | `→ MvuData` | 读取 MVU 变量数据 |
| `Mvu.replaceMvuData(newData, {type, message_id})` | `async` | 写回 MVU 变量（不改消息正文）｜ medium·单源孤证（无官方文档） |
| `Mvu.parseMessage(rawStr, oldData)` | `async → MvuData` | 从含 `<UpdateVariable>` 的原始文本解析变量更新 ｜ medium·单源孤证 |

### 5.3 防抖刷新调度器（schedulePopulate）

**核心结论**：`schedulePopulate` + 数据签名机制防止短时间内频繁重渲染。

```javascript
var lastPopulateSignature = '';
var populateTimer = null;
var populateRaf = null;

function schedulePopulate(delay, force) {
  if (populateTimer) clearTimeout(populateTimer);
  var raf = window.requestAnimationFrame || function(cb) { return setTimeout(cb, 16); };
  var caf = window.cancelAnimationFrame || clearTimeout;
  populateTimer = setTimeout(function() {
    if (populateRaf) caf(populateRaf);
    populateRaf = raf(function() {
      populateRaf = null;
      populate(null, { force: !!force });
      updateAllWideThresholdGrids();
    });
  }, delay || 50);
}

function populate(all, options) {
  options = options || {};
  all = all && hasStatData(all) ? all : readCurrentHudVariables();
  if (hasStatData(all)) lastGoodHudData = cloneHudValue(all);

  // 签名：stat_data + media + settings 三者联合签名
  var sig = JSON.stringify({ stat: statRoot(all), media: getMediaData(), settings: settings });
  if (!options.force && sig === lastPopulateSignature) return;  // 签名未变则跳过
  lastPopulateSignature = sig;

  // 活跃 tab 立即同步渲染，其余 tabs requestIdleCallback 延迟渲染
  var activeId = activeTabId();
  renderTabIfNeeded(activeId, all, sig, true);
  renderAllTabsDeferred(all, activeId);
}
```

**注意**：用户触发操作后若数据已更新但签名缓存未变，须传 `{ force: true }` 强制刷新，否则 UI 不刷新（见避坑 7.11）。

### 5.6 多 iframe 去重机制

**核心结论**：每次楼层重渲染（swipe、edit）都会注入新 iframe。去重机制**优先保留当前 iframe 自身**，只有当自身不在 hudFrames 列表中时才降级为保留最后一个，并移除其余。

```javascript
function normalizeDuplicateHudInMessage() {
  var frame = window.frameElement;
  var message = frame && frame.closest ? frame.closest('.mes') : null;
  if (!frame || !message) return 0;

  var frames = Array.prototype.slice.call(message.querySelectorAll('iframe'));
  var hudFrames = frames.filter(function(item) {
    var doc = item.contentDocument;
    return !!(doc && doc.querySelector('.hud-container .hud-title'));
  });

  if (hudFrames.length <= 1) return 0;

  var keep = frame;  // 优先保留自身
  if (hudFrames.indexOf(keep) < 0) keep = hudFrames[hudFrames.length - 1];
  hudFrames.forEach(function(item) {
    if (item !== keep) item.remove();
  });
}

  const nextData = await Mvu.parseMessage(raw, oldData);
  if (!nextData) throw new Error('未能解析变量更新');

  await Mvu.replaceMvuData(nextData, { type: 'message', message_id: 'latest' });
}
```

### 5.11 custom- 前缀对策——MutationObserver 动态补齐

**核心结论**：检测到 `custom-` 前缀存在后，用 MutationObserver 实时给新增节点补全 `custom-` 版本类名。

```javascript
function applyCustomPrefix(root) {
  const XY_PX = /(^|\s)custom-xy-/.test(root.className || '') ? 'custom-' : '';
  if (!XY_PX) return; // 离线预览模式，无需处理

  const prefixEl = (el) => {
    if (!el || el.nodeType !== 1 || !el.className) return;
    el.className.split(/\s+/).forEach(token => {
      if (!token || token.indexOf(XY_PX) === 0) return;
      const prefixed = XY_PX + token;
      if (!el.classList.contains(prefixed)) el.classList.add(prefixed);
    });
  };
  const prefixTree = (el) => {
    prefixEl(el);
    if (el.querySelectorAll) el.querySelectorAll('[class]').forEach(prefixEl);
  };

  prefixTree(root);  // 处理现有节点

  if (!root.__xyPrefixObs && typeof MutationObserver === 'function') {
    root.__xyPrefixObs = new MutationObserver(muts => {
      muts.forEach(m => {
        if (m.type === 'attributes') { prefixEl(m.target); return; }
        m.addedNodes.forEach(n => { if (n.nodeType === 1) prefixTree(n); });
      });
    });
    root.__xyPrefixObs.observe(root, {
      childList: true, subtree: true,
      attributes: true, attributeFilter: ['class'],
    });
  }
}
```

来源：`control_center.js` v2.9.9 第 1758-1789 行 ｜ 置信度：high `[运行时源码]`

## 7. 高频避坑

### 7.1 custom- 前缀导致动态注入内容丢样式

**来源**：`public/scripts/chats.js` DOMPurify 钩子直读 + `control_center.js` v2.9.9 ｜ 置信度：high `[运行时源码]`

**机制（已回源 `public/scripts/chats.js` `addDOMPurifyHooks` / `encodeStyleTags` / `decodeStyleTags` 确认）**：
- **`class` 属性**：ST DOMPurify 钩子对每个 class token 加 `custom-` 前缀（`fa-`、`note-` 开头及 `monospace` 豁免）。
- **`<style>` 标签**：被替换为 `<custom-style>编码内容</custom-style>`，CSS 选择器中的 class 名加 `custom-` 前缀；**`id` 选择器不受前缀处理**。
- **`style=""` 属性**：未观察到被重命名为 `custom-style=""`；是否保留受 ST/DOMPurify 版本与配置影响，不能写成全版本“必然剥除”。
- **JS 注入**：不经过消息静态 class 前缀管线，但仍必须区分受信本地模板与不可信远程 HTML。

**对策（按可靠性排序）**：
1. **受信本地控制面板**：可由本地脚本注入受审模板，CSS 写裸类名；禁止把用户/模型文本直接拼入 HTML。
2. **远程开局页面**：必须走 C10 的“完整性验证 → DOM 净化 → 结构校验 → 注入 → 本地 binder”，不能直接 `innerHTML = remoteHtml`。
3. **消息静态内容**：使用已核 `<style>`/class 规则，或 MutationObserver 动态补齐（见范式 5.11）。
4. **宿主面板兜底**：使用稳定 ID 限定作用域（`#my-panel .xy-row { ... }`）。

### 7.2 整面板空白（分区渲染抛错未隔离）

**来源**：`control_center.js` v2.5.0 vs v2.9.9 对比 ｜ 置信度：high `[运行时源码]`

panel.hidden = !panelOpen;  // 必须在 innerHTML 之后
```
