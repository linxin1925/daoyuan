## 0. 置信度分级体系（全手册适用）

> **收集日期统一声明**：本手册全部条目收集于 **2026-06-28**，交叉验证跨 14 个独立来源。源码级断言的回源核实基于本地 **SillyTavern v1.14.0** + 本地成熟卡（**星月 v2.5.0 / 交错宙域 v2.6.0**）；ST 源码引用的 PR/commit 号为采集时快照，行号可能随版本漂移。

每条结论采用 **置信度等级 + 依据类型** 双轴标注。

### 1 置信度三级

| 级别 | 含义 | 判定标准 |
|---|---|---|
| **high** | 已坐实，可直接施工 | 有运行时证据（真机实测 / 读到运行时实现源码 / 生产卡验证）；结论是源码/文档的直接事实而非下游推导；不在悬案清单 |
| **medium** | 待验证，施工前须实测 | 强制二选一子标签：`medium·单源孤证`（单一来源、无第二源、无实测）/ `medium·推导未验证`（源码或逻辑推导成立但未跑通，须随附真机验证路径）|
| **low** | 孤证无源，禁止据此施工 | 零来源猜测 / 纯历史快照（须标年份 + 窄缩适用范围）|

### 2 依据类型标签（与置信度正交，强制标注）

`[运行时源码]`、`[真机]`、`[文档·多源]`、`[文档·单源]`、`[社区]`、`[推断]`、`[历史孤证:年份]`、`[无源·待考]`

### 3 两条铁规

1. **「从源码事实再推一步」的下游推导，即使源码本身已直读，该推导结论最高只能 `medium·推导未验证`**——除非已真机验证；源码内部事实仍标 `high [运行时源码]`。
2. **节级标注 = 节内条目下确界**——禁止整节标 high 却内含悬案条目；冲突即在节内局部降级，并交叉引用第 13 章。

## 2. ST 服务端：PNG tEXt Chunk 读写管道

### 2.1 读取管道（三段缓存）

**核心结论**：ST 角色卡读取走**三级缓存穿透**链路，命中越早则越快。

```
请求命中 MemoryLimitedMap（内存 LRU，默认 100 MB）
  ↓ 未命中
DiskCache（node-persist 磁盘缓存，5 分钟 SYNC_INTERVAL）
  ↓ 未命中
从 PNG 文件提取 tEXt chunk（character-card-parser）→ 写双层缓存
```

来源：DeepWiki ST 5.1 角色管理 ｜ 适用版本：v1.14.0 ｜ 置信度：high `[运行时源码]`

### 2.2 写入管道

写入时：

1. 调用 `writeCharacterData`
2. `applyAvatarCropResize` 将头像标准化到 **512×768**（Jimp 解码 + 缩放；源码常量 `AVATAR_WIDTH=512, AVATAR_HEIGHT=768`，非 400×600）
3. `write-file-atomic` 原子写入，防止数据损坏

**V2/V3 双写兼容**：写入时同时写 `chara`（CCv2 base64）和 `ccv3`（CCv3，将 spec 改为 `chara_card_v3` 后编码）两个 tEXt chunk，保持向后兼容；读取时 ccv3 优先。V3 写入在 try-catch 内，JSON 解析失败时静默跳过。

来源：`src/character-card-parser.js` write()/read() 函数 + `src/constants.js` AVATAR_WIDTH/AVATAR_HEIGHT ｜ 适用版本：v1.14.0 ｜ 置信度：high `[运行时源码]`

### 2.3 缓存容量规划

经验估算（非官方承诺）：**每 3000 个角色卡 +100 MB 内存缓存**。默认 100 MB 在大规模部署下容易触发 LRU 淘汰，导致频繁回落到磁盘/PNG 解析。

来源：ST 官方文档 config.yaml ｜ 适用版本：v1.14.0 ｜ 置信度：medium·单源孤证 `[文档·单源]`

## 4. 导入堆溢出与防护

### 核心结论

**已知错误**（Issue #3406）：`FATAL ERROR: invalid table size Allocation failed - JavaScript heap out of memory`

复现环境：Node v22.11.0，Windows 10，Chromium 131，卡体积 26 MB。

**根本缺口**（截至 2026-06-28，部分修复状态）：

| 缺口 | 状态 |
|---|---|
| 无预导入文件大小校验 | 未修复 |
| 无图片分辨率约束（Jimp 解码高分辨率图时撑爆堆） | 未修复 |
| 无分块/流式处理大文件路径 | 未修复 |
| Node.js 默认堆上限不足 | 可通过启动参数缓解 |

来源：GitHub Issue #3406 ｜ 适用版本：v1.13.x 前后 ｜ 置信度：high `[文档·单源]`（Issue 复现过程已记录）

### 4.1 实用缓解策略

**构建侧（不依赖 ST 默认保护）**：

- 头像分辨率限制在 **512×768** 以内（对应 `applyAvatarCropResize` 使用的常量 `AVATAR_WIDTH=512, AVATAR_HEIGHT=768`；原文 400×600 有误，已由 `src/constants.js` 源码直读校正）
- JSON 内容最小化（minify），移除开发用注释和多余空白
- 世界书条目文本内联压缩（PList 格式比自然语言紧凑）

**服务端侧**：

- 增大 Node.js 堆上限：`node --max-old-space-size=4096 server.js`
- 配置反向代理（Nginx）限制最大请求体，拦截超大上传

置信度：medium·推导未验证 `[推断]`（构建侧控制分辨率有理论依据，但针对堆溢出的直接效果未实测验证，见第 13 章 U-2）

## 5. 世界书 Token 预算机制

### 5.1 激活与注入顺序

**核心结论**：注入优先级顺序（高→低）：

1. **Constant 类型**条目（无条件注入，优先消耗 Budget）
2. **直接关键词命中**的条目（Order 值大的先注入）
3. **递归扫描**命中的条目

**Budget 耗尽即截止**：一旦 token 预算用完，后续条目不激活，即使关键词命中。

来源：ST 官方文档 World Info ｜ 适用版本：v1.12.x+ ｜ 置信度：high `[文档·多源]`

### 5.2 关键配置参数

| 参数 | 作用 | 性能影响 |
|---|---|---|
| Scan Depth | 向前扫描消息数（默认 2） | 越大 → 匹配越多 → 扫描开销越大 |
| Min Activations | 最低激活数（忽略 Depth 限制） | 配合 Max Depth 避免空扫 |
| Recursion Steps | 递归扫描轮数（0=无限） | 高递归 + 大 Lorebook = 严重卡顿 |
| Context % / Token Budget | Token 配额 | 建议固定 Budget 而非 % |
| Case-sensitive | 大小写精确匹配 | 启用可减少误触 |
| Match whole words | 全词匹配 | 启用可减少误触 |

来源：ST 官方文档 World Info ｜ 适用版本：v1.12.x+ ｜ 置信度：high `[文档·多源]`

### 5.3 Token 量化建议

| 建议 | 说明 | 置信度 |
|---|---|---|
| PList 碎片化条目 + 递归比单个大条目更省 token | 按需激活，未命中时零开销 | medium·单源孤证 `[社区]` |
| Non-Recursable 标记阻断级联注入 | 阻止该条目触发进一步递归 | high `[文档·多源]` |
| Scan Depth=2 足够大多数场景；战斗场景可升至 15 | 社区经验值 | medium·单源孤证 `[社区]` |
| Constant 条目 ≤5 条 | 过多则过早耗尽 Budget，挤出高优先级条目 | high `[文档·多源]` |

优先级框架（社区建议，非官方）：核心身份/铁律 → P9-10；常规背景 → P5-8；稀有细节 → P1-4

来源：World Info Encyclopedia (rentry.co) ｜ 置信度：medium·单源孤证 `[社区]`

## 6. 楼层 iframe 渲染性能

### 6.1 渲染节流总体架构

**核心结论**：交错宙域 v2.6.0 生产状态栏实现了**双缓冲节流**架构：

```
MVU 变量事件 → schedulePopulate(80ms 去抖)
  → setTimeout(delay) 延迟
    → requestAnimationFrame(帧同步)
      → populate()
        → 签名比对（相同则跳过）
        → 当前 tab 立即渲染
        → 其余 tab 分批 IdleCallback 延迟渲染
```

关键去抖时间：
- MVU 事件触发：80ms 去抖（`VARIABLE_UPDATE_ENDED` / `VARIABLE_INITIALIZED`）
- 默认 `schedulePopulate` 延迟：50ms
- IdleCallback 超时上限：180ms
- IdleCallback fallback（不支持时）：setTimeout 45ms
- 每批延迟渲染 tab 数：2 个

来源：交错宙域 v2.6.0 `status_bar_regex_r6.html` ｜ 适用版本：交错宙域 v2.6.0 ｜ 置信度：high `[真机]`

### 6.2 签名比对跳过无效渲染

**核心结论**：签名仅序列化**顶层摘要对象**（`statRoot`），而非完整变量树，避免深层 `JSON.stringify` 开销。签名相同时直接返回，不触发任何 DOM 操作。

关键设计取舍：签名包含 `statRoot(all)` + 媒体状态摘要 + 设置项，不含完整 data blob。

来源：交错宙域 v2.6.0 `status_bar_regex_r6.html` ｜ 适用版本：交错宙域 v2.6.0 ｜ 置信度：high `[真机]`

### 6.3 折叠状态完全跳过渲染

**核心结论**：HUD 折叠时，`renderAllTabsDeferred` 在入口直接 `return`，并在每批 nextBatch 执行前再次检查折叠状态，确保已排队的批次也能及时取消。

```javascript
if ($('#hud-body').hasClass('collapsed')) return;  // 入口检查
// nextBatch 内再次检查：
if (isCollapsed()) { cancelDeferredTabRender(); return; }
```

来源：交错宙域 v2.6.0 `status_bar_regex_r6.html` ｜ 适用版本：交错宙域 v2.6.0 ｜ 置信度：high `[真机]`

### 6.4 扩展 Hook 5 秒超时

**核心结论（已校准范围）**：ST 扩展**生命周期 hook 函数**（`activate/install/update/delete/enable/disable/clean`）支持返回 Promise，但有 **5 秒超时**硬限制。超时后警告继续执行（不阻塞主流）。**注意：此 5 秒限制作用于扩展 manifest hook，不是普通 `eventOn()` 监听器，也不直接适用于 Tavern Helper eval 路径**（eval 路径走不同执行通道，见第 13 章 U-3）。

**大卡 eval 痛点（降级）**：状态栏 HTML 体积 >100 KB 时，Tavern Helper eval 是否受同一 5s 限制约束，目前无源码直证，见第 13 章 U-3。

**规避策略**（medium·推导未验证，需实测）：
- Hook 内只做轻量调度，不做重计算
- 重计算移到 `requestIdleCallback` 或独立 setTimeout 链
- 状态栏打包为**正则嵌入**（注入到消息末尾 DOM），而非走 eval 路径

来源：ST 官方文档扩展开发（lifecycle hook 超时条款） ｜ 适用版本：v1.14.0 ｜ 置信度：medium·单源孤证 `[文档·单源]`（5s 超时仅见于官方文档单一来源，v1.14.0 源码内未直接检索到硬编码 5000ms hook timeout；eval 路径适用性，见第 13 章 U-3）

### 6.5 Tavern Helper iframe 事件体系

**核心结论**：iframe 与 ST 主体通过 postMessage 通信，事件注册用 `eventOn()`/`eventOnce()`，清理用 `eventRemoveListener()`。

关键事件速查：

| 事件 | 含义 |
|---|---|
| `MESSAGE_IFRAME_RENDER_STARTED` | 楼层 iframe 开始渲染 |
| `MESSAGE_IFRAME_RENDER_ENDED` | 楼层 iframe 结束渲染 |
| `GENERATION_STARTED` | 开始生成 |
| `STREAM_TOKEN_RECEIVED_FULLY` | 完整流式 token 到达 |
| `GENERATION_ENDED` | 生成完毕 |
| `VARIABLE_UPDATE_ENDED` | MVU 变量写入完成 |
| `VARIABLE_INITIALIZED` | MVU 变量初始化完成 |

**内存泄漏防范（已校正）**：`eventOn()` 持续监听；根据 JS-Slash-Runner `@types/iframe/event.d.ts` 注释，**当 `eventOn` 所在的前端界面/脚本关闭时，监听将会自动卸载，不必手动调用 `eventRemoveListener` 来移除**。但状态栏 HTML 作为长期存活的 iframe（不会频繁关闭），若在楼层重渲染回调内部动态注册而不清理，仍有累积风险；建议楼层级动态 hook 用完后主动 `eventRemoveListener()`，常驻 HUD 级监听可依赖自动卸载。

来源：JS-Slash-Runner 本地 `@types/iframe/event.d.ts` 直读 ｜ 适用版本：Tavern Helper v4.x（本地 v1.14.0 扩展目录） ｜ 置信度：high `[运行时源码]`（事件体系）+ medium·推导未验证（楼层动态 hook 累积路径，见第 13 章 U-3）

## 7. 变量树膨胀与近期事件压缩

### 7.1 近期事件压缩规则

**核心结论**（生产规约）：

- **压缩触发阈值**：近期事件条目 **>20 条**
- **压缩目标**：≤10 条 + 生成/更新阶段归档条目
- **执行方式**：模型在回复末尾 `<UpdateVariable>` 块中用 JSONPatch 操作（`op=remove` 删低重要度条目，`op=add` 写入归档摘要）

控制中心侧 `persistRecentEventLimit` 开关（I/O 级别），默认 `true`（启用记忆）。

来源：交错宙域 v2.6.0 `mvu_update_compact.txt` 第 42 行 ｜ 适用版本：交错宙域 v2.6.0 ｜ 置信度：high `[真机]`

### 7.2 EJS Context 字段截断

**核心结论**：`mvu_context.ejs` 对所有字符串字段应用 `__czCtxShort(text, limit)` 截断，世界书注入的 `[mvu_context]` 条目是压缩视图，不是完整变量树。

各字段截断上限：

| 字段类型 | 字符上限 |
|---|---|
| 最后互动摘要 | 90 |
| 物品效果 | 70 |
| 天赋/技能效果 | 90 |
| 环境描述 | 90 |
| 身份背景 | 100 |
| 任务描述 | 80 |
| 近期事件摘要 | 100 |

Record 类型容器默认展示条数上限：`__czCtxRecordSummary(path, mapper, limit=8)`，每类最多 8 条。

来源：交错宙域 v2.6.0 `mvu_context.ejs` ｜ 适用版本：交错宙域 v2.6.0 ｜ 置信度：high `[真机]`

### 7.3 摘要更新策略

**核心结论**：摘要更新为**辅助字段维护**策略——不新增顶层容器，仅在剧情/状态/位置/任务等可见变化时，顺手更新已有对象内的 `摘要/备注/描述` 类字段。

`summaryUpdateEnabled` 可从控制中心开关；关闭后对应注入 prompt 不生成，减少模型 overhead。

来源：交错宙域 v2.6.0 `control_center.js` 第 1529 行 ｜ 适用版本：交错宙域 v2.6.0 ｜ 置信度：high `[真机]`

## 9. 正则脚本性能

### 9.1 正则列表管理卡顿

## 10. 资源懒加载与离线 Vendor

### 10.1 无 CDN 依赖原则

**核心结论**：交错宙域 v2.6.0 状态栏 HTML 内**无任何外部 `<script src>` 或 CDN 引用**。jQuery 和 Lodash 通过 Tavern Helper（js-slash-runner）全局注入，iframe 内直接使用 `window.$` / `window._`，无 CDN 请求。

**性能意义**：ST 初始化卡死 30-45s（Issue #4290）的一个已知因素是外部连接超时。Vendor 从 Tavern Helper 扩展本地取，完全规避此风险。

来源：交错宙域 v2.6.0 `status_bar_regex_r6.html` + GitHub Issue #4290 ｜ 置信度：high `[真机]`

### 10.2 媒体库懒加载

**核心结论**：`media_library.js` 使用 `summaryOf()` 返回轻量状态摘要（不含完整 blob）；签名比对时只传摘要对象；图片仅在弹出预览时通过 fetch 懒加载。

来源：交错宙域 v2.6.0 `media_library.js` ｜ 适用版本：交错宙域 v2.6.0 ｜ 置信度：high `[真机]`

## 11. 高频避坑清单

| 坑 | 影响 | 规避 | 置信度 |
|---|---|---|---|
| 头像分辨率过高 | 导入堆溢出 OOM | 构建时限 **512×768**（ST 标准化尺寸，源码 AVATAR_WIDTH/HEIGHT 直证；原文 400×600 有误） | medium·推导未验证 |
| `lazyLoadCharacters=true` + 第三方扩展 | 扩展读浅对象报错 | 测试兼容后再开；模糊搜索必然失效 | high |
| Hook 超时 5s | 状态栏注入逻辑被截断 | Hook 只做轻量调度，重操作异步化 | high（超时存在）/ medium（规避效果） |
| 正则全文扫描大 HTML | 每楼耗时翻倍 | 最小范围匹配，锚点定位 | high |
| 近期事件不压缩 | 变量树膨胀，世界书条目超 token | 规则强制 >20 条压缩到 ≤10 条 | high |
| EJS Context 不截断 | 世界书注入 token 失控 | 所有字段走 `__czCtxShort(text, limit)` | high |
| `saveSettingsDebounced()` 漏调 | 频繁磁盘 I/O 抢占 | 遵循 ST 扩展规范 debounced 写 | medium·单源孤证 |
| Constant WI 条目过多 | Budget 早耗尽，高优先级条目进不来 | Constant 条目 ≤5 条 | high |
| CDN 依赖 | 防火墙/网络问题导致 30-45s 初始化卡死 | Vendor 从 Tavern Helper 本地取，不引外部 URL | high |
| HUD 折叠时继续渲染 | 白耗 CPU | 折叠时 `cancelDeferredTabRender()` + 跳出 | high |
| `eventOn()` 不清理 | 楼层重渲染内动态注册时可能累积监听器 | 常驻 HUD 级监听可依赖 JS-Slash-Runner 自动卸载（前端界面关闭时）；楼层级动态 hook 仍建议主动 `eventRemoveListener()` | medium·推导未验证（自动卸载源码路径未追踪到底层实现；另见第 13 章 U-3） |

## 12. 完整可用范式代码

### 范式 A：状态栏渲染节流（双缓冲 + 签名跳过）

来源：`status_bar_regex_r6.html` ｜ 置信度：high `[真机]`

```javascript
var lastPopulateSignature = '';
var populateTimer = null;
var populateRaf = null;

// 签名只含顶层摘要，不含媒体 blob，避免深层 stringify 开销
function hudDataSignature(all) {
  try {
    return JSON.stringify({
      stat: statRoot(all),
      settings: { key1: val1 }  // 影响渲染的设置项
    });
  } catch (e) { return String(Date.now()); }
}

// delay 默认 50ms；MVU 事件触发时传 80ms
function schedulePopulate(delay, force) {
  if (populateTimer) clearTimeout(populateTimer);
  var raf = window.requestAnimationFrame || function (cb) { return setTimeout(cb, 16); };
  var caf = window.cancelAnimationFrame || clearTimeout;
  var run = function () {
    if (populateRaf) caf(populateRaf);
    populateRaf = raf(function () {
      populateRaf = null;
      populate(null, { force: !!force });
    });
  };
  populateTimer = setTimeout(run, delay || 50);
}

function populate(all, options) {
  options = options || {};
  all = all || readCurrentData();
  var sig = hudDataSignature(all);
  if (!options.force && sig === lastPopulateSignature) return; // 无变化直接跳出
  lastPopulateSignature = sig;
  renderActiveTab(all);           // 当前 tab 立即渲染
  renderOtherTabsDeferred(all);   // 其余 tab 延迟批量渲染
}

// MVU 事件挂载（init 阶段）
eventOn(Mvu.events.VARIABLE_UPDATE_ENDED, function () { schedulePopulate(80); });
eventOn(Mvu.events.VARIABLE_INITIALIZED, function () { schedulePopulate(80); });
```

### 范式 B：Tab 分批 IdleCallback 延迟渲染

来源：`status_bar_regex_r6.html` ｜ 置信度：high `[真机]`

```javascript
var deferredRenderHandle = null;
var deferredRenderUsesIdle = false;

function cancelDeferredTabRender() {
  if (!deferredRenderHandle) return;
  if (deferredRenderUsesIdle && window.cancelIdleCallback)
    window.cancelIdleCallback(deferredRenderHandle);
  else clearTimeout(deferredRenderHandle);
  deferredRenderHandle = null;
}

function scheduleDeferredTabRender(callback) {
  if (window.requestIdleCallback) {
    deferredRenderUsesIdle = true;
    deferredRenderHandle = window.requestIdleCallback(callback, { timeout: 180 });
  } else {
    deferredRenderUsesIdle = false;
    deferredRenderHandle = setTimeout(callback, 45);
  }
}

function renderOtherTabsDeferred(all) {
  cancelDeferredTabRender();
  if (isCollapsed()) return;       // 折叠时直接返回，不排队
  var ids = getNonActiveTabIds();
  var i = 0;
  function nextBatch() {
    if (isCollapsed()) { cancelDeferredTabRender(); return; }  // 批次间再次检查
    var count = 0;
    while (i < ids.length && count < 2) {   // 每批最多 2 个 tab
      renderTabIfNeeded(ids[i++], all);
      count++;
    }
    if (i < ids.length) scheduleDeferredTabRender(nextBatch);
  }
  scheduleDeferredTabRender(nextBatch);
}
```

### 范式 D：EJS 上下文截断工具函数

来源：交错宙域 v2.6.0 `mvu_context.ejs` ｜ 置信度：high `[真机]`

```javascript
// 字段截断：超出 limit 字符则加省略号
const __czCtxShort = (value, limit = 80) => {
  const text = String(value || '').replace(/\s+/g, ' ').trim();
  return text.length > limit ? text.slice(0, limit) + '...' : text;
};

// Record 展示条数限制（默认每类最多 8 条）
const __czCtxRecordSummary = (path, mapper, limit = 8) =>
  Object.fromEntries(
    getEntries(path).slice(0, limit).map(([name, data]) => [name, mapper(data, name)])
  );
```
