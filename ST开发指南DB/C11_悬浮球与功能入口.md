## 1. 定义与边界

### 1.1 什么是“功能入口”

功能入口是宿主界面中的 **launcher**。它不自己实现人物面板、状态栏、控制中心、开局页或媒体库，只把用户意图转换成稳定命令：

```text
Wand / 悬浮球 / 页面按钮 / 快捷键
                 ↓
          command registry
                 ↓
     openHud / openNpc / openWorkshop ...
```

因此入口层的验收标准不是“面板内容是否丰富”，而是：

- 入口在正确的宿主文档中只存在一份；
- 每个动作只触发一次；
- 不可用功能有明确禁用或说明；
- 换入口不会复制业务处理器；
- 切聊天、热重载、切卡后不会残留旧节点和旧监听。

### 1.2 四个容易混淆的概念

| 概念 | 职责 | 本册是否负责 |
|---|---|---:|
| Wand / 魔棒按钮 | 使用 ST/酒馆助手既有菜单作为入口 | 只讲统一路由；挂载细节转 E5 |
| 悬浮球 | 可移动、可吸边的常驻 launcher | 是 |
| 抽屉式状态栏 | 从视口边缘展开的 HUD 外壳 | 否，见 C12 |
| 浮窗式状态栏 | 可拖动、可缩放的 HUD 外壳 | 否，见 C13 |

“悬浮球打开浮窗”不代表二者是同一种组件。悬浮球消失时，已经打开的浮窗仍可独立存在；浮窗失败时，悬浮球也不应被拖死。

## 2. 总体架构

推荐把“入口、命令、外壳、内容”拆成四层：

```text
EntryAdapter
  ├─ WandEntry
  ├─ OrbEntry
  └─ PageButtonEntry
          ↓ dispatch(actionId)
CommandRegistry
          ↓
HudHostCore / OtherFeatureHost
          ↓
DrawerShell / FloatingShell / ModalShell
          ↓
Feature body / iframe / 状态栏真身
```

这套分层解决三个常见退化：

1. Wand、悬浮球各写一套 `if (action === ...)`，日后行为不一致；
2. 入口脚本直接引用另一个闭包里的私有函数，点击时才出现 `ReferenceError`；
3. 功能内容重载时入口也被重复创建，最后一击触发两次。

入口层只能依赖公开命令，不应跨闭包调用未导出的页面内部函数。

## 3. 入口注册表

### 3.1 最小契约

```javascript
const commandRegistry = new Map();

export function registerCommand(definition) {
  const { id, label, icon, available, handler } = definition;
  if (!id || typeof handler !== 'function') {
    throw new TypeError('command requires id and handler');
  }
  if (commandRegistry.has(id)) {
    throw new Error(`duplicate command: ${id}`);
  }
  const availability = typeof available === 'function'
    ? available
    : () => available === false ? '功能已禁用' : true;
  commandRegistry.set(id, {
    id,
    label: label || id,
    icon: icon || '',
    available: availability,
    handler,
  });
  return () => commandRegistry.delete(id);
}

export async function dispatchCommand(id, context = {}) {
  const command = commandRegistry.get(id);
  if (!command) throw new Error(`unknown command: ${id}`);
  const result = await command.available(context);
  if (result !== true) {
    const reason = typeof result === 'string' ? result : '当前功能不可用';
    throw new Error(reason);
  }
  return command.handler(context);
}
```

`available` 不只返回布尔值，最好允许返回原因。入口适配器可以把原因渲染为 `disabled`、tooltip 或 toast，而不是点击后静默无响应。

### 3.2 单一命令源

推荐命令定义：

```javascript
const unregisterHud = registerCommand({
  id: 'hud.toggle',
  label: '状态栏',
  icon: 'fa-chart-simple',
  available: () => hostWindow().CardRuntime?.hudReady
    ? true
    : '状态栏运行时尚未就绪',
  handler: ({ source }) => hostWindow().CardRuntime.hud.toggle({ source }),
});
```

三个入口只传来源：

```javascript
dispatchCommand('hud.toggle', { source: 'wand' });
dispatchCommand('hud.toggle', { source: 'orb' });
dispatchCommand('hud.toggle', { source: 'opening-page' });
```

来源字段用于日志与埋点，不应用来分叉业务语义。确需不同能力时，注册不同命令 ID。

### 3.3 入口健康状态

每次打开菜单前重新计算 `available()`，不要在初始化时永久缓存。聊天、卡片、依赖和生成状态都可能变化。

建议至少区分：

- `ready`：可直接触发；
- `busy`：正在加载或生成，暂时禁用；
- `missing_dependency`：缺少 Tavern Helper、MVU 或远程资源；
- `wrong_context`：当前不是目标卡/目标聊天；
- `failed`：上次初始化失败，允许重试；
- `hidden`：当前设备或配置不展示该入口。

## 4. 悬浮球状态机

### 4.1 推荐状态

```text
unmounted
  → free-idle / docked-idle
  → pointerdown
      ├─ 位移超过阈值 → dragging → settle → free / docked
      ├─ 吸附态长按 → half-hidden / hidden
      └─ 未移动 → menu-open
  → action-dispatch → menu-close
  → destroyed
```

运行态建议至少保存：

```javascript
const orbState = {
  side: 'right',
  top: 0.5,
  open: false,
  dockDepth: 'full',
  fx: 0.92,
  fy: 0.55,
  docked: true,
};
```

`fx/fy/top` 使用相对视口的归一化坐标，便于在桌面、窄屏和旋转后恢复；实际绘制前仍须重新夹取。

### 4.2 点击与拖动仲裁

不要依靠浏览器自动产生的 `click` 判断。一次指针序列中明确记录：

- `pointerId`；
- 起点与当前点；
- 是否已超过拖动阈值；
- 长按计时器；
- 是否发生 `pointercancel`；
- 是否已经 dispatch。

```javascript
const DRAG_THRESHOLD = 5;
let gesture = null;

function onPointerDown(event) {
  if (!event.isPrimary || (event.pointerType === 'mouse' && event.button !== 0)) return;
  if (gesture) return;
  event.currentTarget.setPointerCapture(event.pointerId);
  gesture = {
    pointerId: event.pointerId,
    startX: event.clientX,
    startY: event.clientY,
    startRect: getOrbRect(),
    dragging: false,
    cancelled: false,
  };
}

function onPointerMove(event) {
  if (!gesture || event.pointerId !== gesture.pointerId) return;
  const distance = Math.hypot(
    event.clientX - gesture.startX,
    event.clientY - gesture.startY,
  );
  if (distance > DRAG_THRESHOLD) gesture.dragging = true;
  if (gesture.dragging) {
    moveOrb(
      gesture.startRect.x + event.clientX - gesture.startX,
      gesture.startRect.y + event.clientY - gesture.startY,
    );
  }
}

function finishPointer(event, cancelled = false) {
  if (!gesture || event.pointerId !== gesture.pointerId) return;
  const wasDrag = gesture.dragging;
  clearLongPressTimer();
  try { event.currentTarget.releasePointerCapture(event.pointerId); } catch (_error) {}
  gesture = null;
  if (cancelled) return;
  if (wasDrag) settleAndPersist();
  else toggleOrbMenu();
}
```

iOS 系统边缘手势、通知中心和浏览器导航可能产生 `pointercancel`。它必须和 `pointerup` 一样清计时器、释放临时态，但不能当作点击。

### 4.3 长按与半隐藏

长按只能在“仍未拖动、仍为同一 pointer、悬浮球处于吸附态”时切换收纳深度。进入拖动、取消、抬起或销毁时都必须取消计时器。

半隐藏不能把可点击面积一并缩到屏幕外。视觉主体可移出一部分，但触控热区仍应至少 44×44 CSS px，并保留可见提示。

## 5. 位置、吸附与持久化

### 5.1 可用视口

优先使用 `visualViewport`，同时保留无该 API 的回退：

```javascript
function hostWindow() {
  try {
    if (window.parent && window.parent !== window && window.parent.document) {
      return window.parent;
    }
  } catch (_error) {}
  return window;
}

function visibleViewport() {
  const host = hostWindow();
  const vv = host.visualViewport;
  return vv
    ? { x: vv.offsetLeft, y: vv.offsetTop, w: vv.width, h: vv.height }
    : { x: 0, y: 0, w: host.innerWidth, h: host.innerHeight };
}
```

入口挂在宿主 document 时，视口、`matchMedia`、`getComputedStyle`、存储和跨重载单例也必须从 `hostWindow()` 取得。裸 `window` 在 TH iframe 中只代表脚本 iframe。

夹取时还应减去：

- `env(safe-area-inset-*)`；
- ST 顶栏或侧栏实际占用；
- 底部输入区；
- 悬浮球自身尺寸与可触控热区。

### 5.2 吸附策略

常见策略：

- 距离左右边缘小于阈值时吸边；
- 四角附近优先保留轮盘展开净空；
- 用户关闭吸附时保存自由坐标；
- 切横竖屏后重新夹取，不直接相信旧像素；
- settle 时只计算一次目标真值并立即持久化；CSS 只从当前视觉位置过渡到目标，动画帧不得反写业务状态。

“靠最近边”不是唯一目标。若右边已有 ST 浮动控件，应允许避让或选择净空更大的边。

### 5.3 存储格式

```javascript
const STORAGE_VERSION = 1;

function saveOrbPlacement(state) {
  hostWindow().localStorage.setItem('my-card:orb-placement', JSON.stringify({
    version: STORAGE_VERSION,
    fx: state.fx,
    fy: state.fy,
    side: state.side,
    dockDepth: state.dockDepth,
    docked: state.docked,
  }));
}
```

读取时必须：

1. `try/catch` 处理坏 JSON；
2. 校验版本和字段类型；
3. 对 `fx/fy` 夹到 `[0,1]`；
4. 无法迁移时回到安全默认值；
5. 不把角色卡可导出的秘密或账户数据放入 `localStorage`。

若位置需要按设备保留，浏览器本地存储即可；若需要随聊天同步，应放聊天级元数据，并明确它会污染存档和多设备合并。

## 6. 轮盘菜单与视口夹取

### 6.1 展开方向

固定一个扇形角度会在四角越界。应根据悬浮球四周净空，从上、下、左、右或对角方向中选择可容纳最多按钮的一组角度。

```text
availableLeft / availableRight
availableTop  / availableBottom
              ↓
chooseArc(centerAngle, radius, itemCount)
              ↓
measure → clamp → render
```

### 6.2 测量纪律

动画中的元素被 `scale()` 后，`getBoundingClientRect()` 会返回视觉缩放尺寸，容易低估按钮宽度。菜单布局测量优先使用稳定布局尺寸：

- `offsetWidth/offsetHeight`；
- 关闭 transform 后的离屏测量；
- 明确的 CSS 变量尺寸。

最终每个按钮的位置仍须独立夹取，不能只保证圆心在屏幕内。

### 6.3 DOM 语义

```html
<button id="card-orb" type="button"
        aria-label="打开角色卡功能菜单"
        aria-controls="card-orb-menu"
        aria-expanded="false"></button>

<div id="card-orb-menu" role="menu" hidden>
  <button type="button" role="menuitem" data-action="hud.toggle">状态栏</button>
</div>
```

不要用无 `tabindex`、无键盘事件的 `div` 假装按钮。菜单打开时同步 `aria-expanded`；关闭后把焦点还给悬浮球。

## 7. 输入、无障碍与移动端

### 7.1 统一 Pointer Events

优先使用 `pointerdown/move/up/cancel` + pointer capture，不要分别维护鼠标和触摸两套状态机。样式需按实际手势设置 `touch-action`，但不要粗暴禁用整个页面的缩放和滚动。

### 7.2 键盘

- `Enter` / `Space`：打开菜单或触发当前按钮；
- `Escape`：关闭菜单并归还焦点；
- 方向键：在轮盘项间移动焦点；
- `Tab`：遵循自然顺序，菜单关闭时隐藏内部焦点；
- 不用只靠颜色表示可用/不可用。

拖动是增强能力，不能成为唯一操作。应提供“重置位置”“固定左侧/右侧”设置，让键盘和辅助技术用户也能恢复入口。

### 7.3 Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  .card-orb,
  .card-orb-menu > button {
    animation: none !important;
    transition-duration: 1ms !important;
  }
}
```

Canvas 桌宠、粒子和周期摇晃都应在 reduced-motion 下停用；不只是缩短 CSS 动画。

## 8. 生命周期与资源清理

### 8.1 单例安装

```javascript
const RUNTIME_KEY = '__MY_CARD_ENTRY_RUNTIME__';
const host = hostWindow();

host[RUNTIME_KEY]?.destroy?.();
const api = installEntryRuntime();
host[RUNTIME_KEY] = api;
```

宿主节点使用稳定 ID，并在安装前主动移除失去所有者的旧节点。只检查 `if (node) return` 不够：旧节点可能仍在，但监听器已经指向被销毁的闭包。

### 8.2 必须清理的资源

- 悬浮球、菜单、样式、提示气泡；
- `requestAnimationFrame`；
- 长按、启动重试、周期提示等 timer；
- `resize`、`orientationchange`、`visualViewport` 监听；
- 文档空白点击监听；
- `ResizeObserver` / `MutationObserver`；
- 命令注册返回的 disposer；
- 仍在执行的异步初始化与 AbortController。

```javascript
function destroy() {
  if (destroyed) return;
  destroyed = true;
  try { abortController?.abort(); } catch (_error) {}
  try { hostWindow().cancelAnimationFrame(rafId); } catch (_error) {}
  try { hostWindow().clearTimeout(longPressTimer); } catch (_error) {}
  while (disposers.length) {
    try { disposers.pop()?.(); } catch (error) { console.warn('entry cleanup failed', error); }
  }
  try { root?.remove(); } catch (_error) {}
  try { style?.remove(); } catch (_error) {}
  try {
    const host = hostWindow();
    if (host[RUNTIME_KEY] === api) delete host[RUNTIME_KEY];
  } catch (_error) {}
}
```

在 `pagehide`、卡片运行时销毁和热重载前执行清理。`CHAT_CHANGED` 是否销毁取决于入口是否卡片专属：卡片专属入口必须复核当前聊天身份，不能在别的卡继续显示。

## 9. 星月生产样本

> 样本状态：`rolecard-diy-workshop/runtime/xingyue/3.4.9/control-center.js`，2026-07-14 本地核验。以下结论是生产实现观察，不是通用 API 保证。

星月样本已经实现：

- 宿主级固定悬浮球、Canvas 视觉与 reduced-motion；
- `side/top/fx/fy/docked/dockDepth` 状态；
- Pointer Events、拖动、吸附、长按和 `pointercancel` 清理；
- 根据四向净空选择扇形菜单；
- 使用稳定布局尺寸测量按钮后再夹取；
- 运行时销毁、`pagehide` 清理和热重载前销毁旧实例；
- 悬浮球路由到状态栏、NPC、控制中心等多个功能。

需要通用指南补强的部分：

- 把项目内 `if/else` 动作分发抽成公开命令注册表；
- 为悬浮球位置补齐 `visualViewport`、旋转后的统一重夹取；
- 补全键盘导航、菜单角色与 ARIA 状态；
- 将具体点阵桌宠、时段文案、粒子参数和 `xingyue-*` ID 留在项目层。

历史失败经验同样重要：跨闭包调用开局页私有函数会在点击时出现 `ReferenceError`；Canvas 自身圆形裁剪会切断拖尾。这两类问题都不能靠吞异常修复。

## 10. 高频错误

| 错误 | 症状 | 修复方向 |
|---|---|---|
| 拖动仍触发点击 | 松手后轮盘误开 | 用统一手势状态机仲裁 |
| `pointercancel` 不清长按 | 系统手势后自动半隐 | 所有终止路径清 timer |
| 动画与坐标写入竞争 | 吸边时抖动、回弹 | settle 阶段单写稳定位置 |
| 固定弧度 | 角落按钮越界 | 按四向净空选弧并逐项夹取 |
| 测动画态 rect | 按钮宽度被低估 | 用稳定布局尺寸测量 |
| 只存像素 | 换屏幕后球消失 | 存归一化坐标，恢复时夹取 |
| 入口调用私有闭包 | 点击才报未定义 | 公开命令注册表 |
| 热重载不销毁 | 两个球、动作触发两次 | 全局单例 + destroy |
| Canvas 裁剪特效 | 拖尾/粒子被切断 | 扩大缓冲区或移除错误裁剪 |
| 只支持鼠标 | 手机无法拖动 | Pointer Events + coarse pointer |
| `div` 假按钮 | 键盘和读屏器不可用 | 原生 button + ARIA |
| 业务塞进入口 | 入口小改牵连整套 HUD | launcher/content surface 分层 |

## 11. 验收矩阵

| 维度 | 必验项 |
|---|---|
| 输入 | 单击、慢拖、快拖、长按、`pointercancel`、鼠标与触摸 |
| 位置 | 中央、四边、四角、吸附/自由、半隐/恢复 |
| 视口 | 320/375/768 宽、横竖屏、浏览器缩放、软键盘 |
| 菜单 | 所有按钮不越界，空白点击收起，每次动作只触发一次 |
| 路由 | Wand、悬浮球、页面按钮调用同一命令并得到同一业务结果 |
| 持久化 | 刷新、切聊天、热重载后恢复；坏 JSON 安全回默认 |
| 依赖 | 缺依赖、加载中、失败重试都有可见状态 |
| 无障碍 | 键盘、焦点、Escape、ARIA、44px 热区、reduced-motion |
| 生命周期 | 切卡、pagehide、重载后无残留节点、rAF、timer、listener |

稳定完成门：上述矩阵通过后，才允许把悬浮球作为唯一显式入口；否则必须保留 Wand 或页面内按钮作为可恢复入口。
